The Userplex API allows you to send events, identify users, and track behavior in your applications. All API requests require authentication using a Bearer token.

## Base URL

All API requests should be made to:

```
https://api.userplex.app
```

## Authentication

Authenticate your requests by including your API key as a Bearer token in the `Authorization` header.

```bash
Authorization: Bearer sk_live_your_api_key
```

You can find your API key in the [Userplex Dashboard](https://app.userplex.app) under **Settings > Developer**.

<Warning>
  Keep your API key secure. Never expose it in client-side code or commit it to version control.
</Warning>

## Quick Examples

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    import Userplex from 'userplex';

    const client = new Userplex({
      apiKey: process.env['USERPLEX_API_KEY'],
    });

    // Send an event
    await client.events.capture({
      name: 'user.signup',
      user_id: 'user_123',
      data: { plan: 'pro' }
    });

    // Identify a user
    await client.users.identify({
      user_id: 'user_123',
      email: 'user@example.com',
      name: 'Jane Doe'
    });
    ```
  </Tab>
  <Tab title="Ruby">
    ```ruby
    require "userplex"

    userplex = Userplex::Client.new(
      api_key: ENV["USERPLEX_API_KEY"]
    )

    # Send an event
    userplex.events.capture(
      name: "user.signup",
      user_id: "user_123",
      data: { plan: "pro" }
    )

    # Identify a user
    userplex.users.identify(
      user_id: "user_123",
      email: "user@example.com",
      name: "Jane Doe"
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # Send an event
    curl -X POST https://api.userplex.app/capture \
      -H "Authorization: Bearer $USERPLEX_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"name": "user.signup", "user_id": "user_123"}'

    # Identify a user
    curl -X POST https://api.userplex.app/identify \
      -H "Authorization: Bearer $USERPLEX_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"user_id": "user_123", "email": "user@example.com"}'
    ```
  </Tab>
</Tabs>

## Response Format

All successful responses return a JSON object with a `success` field:

```json
{
  "success": true
}
```

Error responses include an `error` field with a human-readable message:

```json
{
  "error": "Missing or invalid API key"
}
```

## Rate Limits

The API allows up to 1,000 requests per minute per API key.
